# 工具实现模块

## 概述
实现六个 MCP 工具并通过 `@tool` 装饰器注册到 registry，作为 MCP 服务器对外暴露的入口点。

## 对外接口

本模块对外接口为六个被 `@tool` 装饰器注册的异步函数，由 `server` 模块通过 `registry` 统一分发：

- `project_map() -> str` — 返回项目级架构概览（缓存命中直接返回，否则返回生成引导）
- `explore_module(module_name: str) -> str` — 返回指定模块的深度架构摘要（含辅助目录降级处理与缓存校验）
- `get_project_map_prompt() -> str` — 返回用于生成项目模块划分的 LLM 提示词
- `submit_project_map(response: str) -> str` — 接收模块划分文本，写入缓存并返回渲染后的架构 Markdown
- `get_module_prompt(module_name: str) -> str` — 返回用于生成指定模块摘要的 LLM 提示词
- `save_module_summary(module_name: str, summary: str) -> str` — 将模块摘要写入缓存

## 内部文件

- `__init__.py` — 导入所有工具子模块以触发 `@tool` 注册，`__all__` 为空（注册副作用为主）
- `project_map.py` — `project_map` 工具实现，校验环境变量与 CodeGraph DB 存在性，缓存有效则返回缓存，否则返回重建引导
- `explore_module.py` — `explore_module` 工具实现，含参数校验、辅助目录降级、模块名匹配、按模块哈希做缓存校验
- `get_project_map_prompt.py` — `get_project_map_prompt` 工具实现，委派 `summarizer` 生成提示词
- `submit_project_map.py` — `submit_project_map` 工具实现，委派 `summarizer` 写入缓存
- `get_module_prompt.py` — `get_module_prompt` 工具实现，委派 `summarizer` 生成模块提示词
- `save_module_summary.py` — `save_module_summary` 工具实现，委派 `summarizer` 保存摘要

## 依赖关系

**上游（本模块依赖）：**
- `src/codesense_v1`（核心基础：`errors.InvalidArgumentError` / `LLMError`）
- `src/codesense_v1/cache`（缓存读写与哈希校验）
- `src/codesense_v1/data`（`CodeGraphDB` 只读查询）
- `src/codesense_v1/registry`（`@tool` 装饰器注册）
- `src/codesense_v1/summarizer`（提示词生成、摘要保存、项目架构提交）

**下游（依赖本模块）：**
- 无（本模块为顶层工具入口，仅由 `server` 通过 `registry` 间接调用）

## 外部依赖

- 标准库：`os`、`pathlib.Path`、`typing.Final`
- 项目内：`codesense_v1`（cache / data / registry / summarizer / errors）

## 设计要点

1. **统一注册模式**：每个工具文件用 `@tool(name, description, input_schema)` 装饰 async 函数，`__init__.py` 仅靠 import 副作用完成注册。
2. **缓存优先**：`project_map` 与 `explore_module` 均先校验缓存（DB 哈希 / 模块哈希），命中直接返回，未命中返回结构化引导文本而非报错。
3. **环境变量驱动**：所有工具从 `CODESENSE_PROJECT_ROOT` 读取项目根，缺失时抛 `InvalidArgumentError`。
4. **错误分层**：参数错误抛 `InvalidArgumentError`，DB 缺失返回友好 Markdown 提示，`summarizer` 层异常向上传播。
5. **辅助目录降级**：`explore_module` 对 L2 辅助目录（tests/scripts 等）返回简化说明，不强制生成完整摘要。
