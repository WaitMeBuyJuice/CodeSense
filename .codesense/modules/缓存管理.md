## 一句话定位
`.codesense/` 目录缓存的读写与失效管理，提供 segment 级、模块级、子模块级缓存。

## 架构简析
单文件核心层（`cache.py`）+ 导出层（`__init__.py`）。`cache.py` 内按职责分四组：哈希与元信息校验、模块级缓存、project_map segment 缓存、子模块文档缓存；私有辅助函数（`_` 前缀）处理路径构造、元信息写入、目录清理与陈旧模块裁剪。`__init__.py` 仅做符号重导出，无逻辑。

## 对外接口
（依据 `__init__.py` 的 `__all__` 导出，即图推导对外接口）

### 哈希与缓存校验
- `db_hash(db_path: Path) -> str` — 计算指定文件的 SHA-256 十六进制摘要；文件不存在时抛 `FileNotFoundError`。
- `is_cache_valid(codesense_dir: Path, current_hash: str) -> bool` — 当 `project_map.json` 存在且其 `db_hash` 与 `current_hash` 一致时返回 `True`，任何错误返回 `False`。
- `is_segment_valid(codesense_dir: Path, segment_id: str, current_hash: str) -> bool` — segment 存在且其存储哈希与 `current_hash` 匹配时返回 `True`。

### project_map 整体读写
- `read_project_map(codesense_dir: Path) -> str | None` — 读取 `project_map.md`，缺失/不可读返回 `None`。
- `write_project_map(codesense_dir: Path, content: str, current_hash: str) -> None` — 写 `project_map.md` 并更新 `project_map.json` 元信息；自动创建目录。
- `render_project_map(codesense_dir: Path) -> str | None` — 拼接全部 segment Markdown 为最终 `project_map.md` 并写回；任一 segment 缺失返回 `None`。

### modules_index 读写
- `read_modules_index(codesense_dir: Path) -> dict[str, object] | None` — 解析 `modules_index.json`，缺失/非 dict 返回 `None`。
- `write_modules_index(codesense_dir: Path, modules: list[dict[str, object]], current_hash: str, aux_dirs: list[dict[str, object]] | None = None) -> None` — 写 `modules_index.json`，裁剪不再存在的模块文件与哈希条目，可选写入 `auxiliary_dirs`，更新元信息。

### 模块级缓存
- `read_module(codesense_dir: Path, module_key_str: str) -> str | None` — 读取 `modules/<module_key>.md`，缺失返回 `None`。
- `write_module(codesense_dir: Path, module_key_str: str, module_name: str, summary: str, current_hash: str, module_content_hash: str = "") -> None` — 写模块摘要文件，可选持久化模块内容哈希，更新元信息。`module_name` 仅保留用于 API 兼容。
- `read_module_hashes(codesense_dir: Path) -> dict[str, str]` — 读取 `modules/.hashes.json`，兼容旧扁平格式与新 `{hash, generated_at}` 格式；错误返回空 dict。
- `write_module_hash(codesense_dir: Path, module_key_str: str, module_hash: str) -> None` — 在 `modules/.hashes.json` 中 upsert 模块哈希与时间戳。

### segment 级缓存
- `read_segment(codesense_dir: Path, segment_id: str) -> str | None` — 读取 segment Markdown，错误返回 `None`。
- `read_segment_hash(codesense_dir: Path, segment_id: str) -> str | None` — 读取 segment 存储哈希，缺失返回 `None`。
- `write_segment(codesense_dir: Path, segment_id: str, content: str, source_hash: str) -> None` — 写 segment Markdown 及其源哈希到 `project_map_segments/`。

### 失效
- `invalidate(codesense_dir: Path) -> None` — 删除 `project_map.md`、`modules_index.json`、`project_map.json` 及整个 `modules/` 目录；静默忽略缺失项，清空全部缓存。
- `invalidate_segments(codesense_dir: Path) -> None` — 删除 `project_map_segments/` 下全部文件。

### 文件名键工具
- `module_key(module_path: str) -> str` — 将模块路径中的 `/`、`\` 替换为 `_`，去首尾空白。
- `safe_key(module_name: str) -> str` — 将文件系统非法字符（`/ \ : * ? " < > |` 及控制字符）替换为 `_`，去首尾 `_`，截断至 100 字符；空结果返回 `_`。

## 文件清单
- `src/codesense_v1/cache/__init__.py` — 纯导出层，从 `cache.py` 重导出 20 个公开符号（见 `__all__`），无业务逻辑。
- `src/codesense_v1/cache/cache.py` — 核心实现层，封装 `.codesense/` 目录下 project_map、modules_index、模块摘要、segment、子模块文档的读写与失效，以及哈希校验与文件名键生成。

## 上下游关系
- **上游（依赖此模块的目录）**：`src/codesense_v1/summarizer`、`src/codesense_v1/tools`、`tests`（置信度 extracted）。
- **下游（此模块依赖的目录）**：无项目内依赖；仅依赖 Python 标准库。

## 实现约束清单
- **读函数容错**：所有 `read_*` 函数捕获全部异常并返回 `None`（或空 dict），调用方应按缓存未命中处理。
- **写函数异常透传**：`write_*` 函数在真实 I/O 失败时抛 `OSError`，不吞异常。
- `db_hash` 在 DB 文件缺失时抛 `FileNotFoundError`，是唯一不静默的读路径。
- `invalidate` / `invalidate_segments` 静默忽略缺失文件与目录，不抛异常。
- `write_modules_index` 仅裁剪不再存在于新索引中的模块 `.md` 与哈希条目，保留存活模块的既有摘要，交由逐模块失效判断是否重新生成。
- `read_module_hashes` 兼容两种历史格式：旧扁平 `str` 值与新 `{hash, generated_at}` dict 值。
- segment 固定为 7 段：`01_identity`、`02_structure`、`03_modules`、`04_constraints`、`05_flows`、`06_concepts`、`07_dependencies`；`render_project_map` 按此顺序拼接，任一缺失即整体失败返回 `None`。
- 子模块文档函数（`submodule_dir` / `read_submodule` / `write_submodule` / `read_submodule_hashes` / `write_submodule_hash`）存在于 `cache.py` 但未纳入 `__init__.py` 的 `__all__`，属模块内部未导出符号。
- 文件名键经 `safe_key` 处理，原始模块名存于 JSON payload 的 `module_name` 字段，键本身仅作文件名。

## 外部依赖
- 标准库：`hashlib`、`json`、`re`、`datetime`、`pathlib`
