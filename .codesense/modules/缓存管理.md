# 模块理解文档 · 缓存管理

## 一句话描述

管理项目地图、模块索引与模块摘要的文件缓存读写与失效。

## 对外接口

> 以下为 `cache.py` 中未以 `_` 开头的公共函数（按 Python 约定视为对外暴露）。

| 函数 | 签名 | 说明 |
|---|---|---|
| `db_hash` | `(db_path: Path) -> str` | 计算数据库文件的哈希值，用于缓存版本校验 |
| `is_cache_valid` | `(codesense_dir: Path, current_hash: str) -> bool` | 比对当前哈希与缓存元数据，判断缓存是否仍有效 |
| `read_project_map` | `(codesense_dir: Path) -> str \| None` | 读取已缓存的项目地图内容；不存在则返回 `None` |
| `write_project_map` | `(codesense_dir: Path, content: str, current_hash: str) -> None` | 写入项目地图内容并更新缓存元数据 |
| `read_modules_index` | `(codesense_dir: Path) -> dict[str, object] \| None` | 读取模块索引缓存（字典形式） |
| `write_modules_index` | `(codesense_dir: Path, modules: list[dict[str, object]], current_hash: str) -> None` | 写入模块索引缓存并更新元数据 |
| `read_module` | `(codesense_dir: Path, module_key_str: str) -> str \| None` | 依据模块键读取单个模块的缓存摘要 |
| `write_module` | `(codesense_dir: Path, module_key_str: str, module_name: str, summary: str, current_hash: str) -> None` | 写入单个模块的摘要缓存 |
| `invalidate` | `(codesense_dir: Path) -> None` | 清除该项目的全部缓存（含元数据与模块目录） |
| `module_key` | `(module_path: str) -> str` | 由模块路径生成缓存键 |
| `safe_key` | `(module_name: str) -> str` | 将模块名转换为文件系统安全的键名 |

**内部辅助函数（以 `_` 开头，非对外接口）：**
`_now_iso`、`_meta_path`、`_write_meta`、`_clear_modules_dir`

## 内部文件

| 文件 | 作用 |
|---|---|
| `src/codesense_v1/cache/__init__.py` | 包初始化文件，当前未声明任何导出符号，需通过 `cache.cache` 子模块访问接口 |
| `src/codesense_v1/cache/cache.py` | 实现全部缓存读写、哈希校验、键名生成与缓存失效逻辑的核心实现文件 |

## 依赖关系

| 方向 | 目录 |
|---|---|
| **上游**（本模块依赖） | 无 |
| **下游**（依赖本模块） | `src/codesense_v1/summarizer`、`tests` |

### 关系解读

- 该模块为**叶子级基础组件**，不依赖项目内其他目录，仅依赖标准库（如 `Path`、`hashlib`、`json` 等），具备良好的可独立测试性。
- **summarizer** 是其主要消费方：在生成项目地图、模块索引与模块摘要前后调用 `is_cache_valid` / `read_*` / `write_*` 实现命中复用或回源重算。
- **tests** 直接对其行为进行验证，印证其作为公共契约的稳定性要求。
