# 模块理解文档：缓存管理

## 1. 一句话描述

提供查询结果与中间数据的缓存读写、校验与失效管理。

## 2. 对外接口

当前 `__init__.py` 未导出符号，模块的公开能力直接位于 `cache.py` 中。以下接口名称均不以 `_` 开头，按 Python 惯例视为对外暴露：

| 接口 | 签名 | 说明 |
|---|---|---|
| `db_hash` | `(db_path: Path) -> str` | 计算数据库文件的哈希，用于缓存版本校验 |
| `is_cache_valid` | `(codesense_dir: Path, current_hash: str) -> bool` | 判断当前缓存是否仍与数据库哈希一致 |
| `read_project_map` | `(codesense_dir: Path) -> str \| None` | 读取已缓存的 project_map 内容 |
| `write_project_map` | `(codesense_dir: Path, content: str, current_hash: str) -> None` | 写入 project_map 内容并更新元数据 |
| `read_modules_index` | `(codesense_dir: Path) -> dict[str, object] \| None` | 读取模块索引缓存 |
| `write_modules_index` | `(codesense_dir: Path, modules: list[dict[str, object]], current_hash: str) -> None` | 写入模块索引缓存 |
| `read_module_hashes` | `(codesense_dir: Path) -> dict[str, str]` | 读取各模块内容的哈希表 |
| `write_module_hash` | `(codesense_dir: Path, module_key_str: str, module_hash: str) -> None` | 写入单个模块的内容哈希 |
| `read_module` | `(codesense_dir: Path, module_key_str: str) -> str \| None` | 读取单个模块的缓存结果 |
| `write_module` | `(codesense_dir, module_key_str, module_name, summary, current_hash, module_content_hash="") -> None` | 写入单个模块的摘要缓存 |
| `invalidate` | `(codesense_dir: Path) -> None` | 使整个缓存失效 |
| `module_key` | `(module_path: str) -> str` | 由模块路径生成缓存键 |
| `safe_key` | `(module_name: str) -> str` | 将模块名转换为安全缓存键 |

> 以下划线开头的 `_now_iso`、`_meta_path`、`_write_meta`、`_clear_modules_dir`、`_prune_stale_modules` 为内部辅助函数，不对外暴露。

## 3. 内部文件

| 文件 | 作用 |
|---|---|
| `src/codesense_v1/cache/__init__.py` | 包初始化文件，当前未导出任何符号 |
| `src/codesense_v1/cache/cache.py` | 实现缓存元数据、索引、模块内容、哈希校验与失效策略的核心逻辑 |
