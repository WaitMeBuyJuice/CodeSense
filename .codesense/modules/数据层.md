# 数据层（Data Layer）

## 一句话定位
CodeGraph SQLite 知识图谱的只读封装层，提供文件/符号/边查询、模块依赖提取与目录级架构特征计算。

## 架构简析
模块内部分三层，全部为纯读层（`CodeGraphDB` 以 `mode=ro` 只读打开 SQLite）：

- **存储边界层**（`db.py`）：唯一与 CodeGraph SQLite 直接交互的文件，对外暴露 `CodeGraphDB` 及三个冻结数据类 `FileRow`/`NodeRow`/`EdgeRow`。schema 变更只需改此文件。
- **数据派生层**：基于 `CodeGraphDB` 的原始行做计算。
  - `modules.py`：文件级依赖边提取（`imports`/`calls` → `ModuleEdge`），是依赖图的核心。
  - `aggregate.py`：文件边聚合到目录视图。
  - `architecture.py`：语言无关的图度量（中心性/分层/环/跨目录公共 API/外部依赖）。
  - `files.py`：文件列表与目录树（合并 `.gitignore`/`.codesenseignore`）。
  - `structure.py`：顶层目录分类（主目录/辅助目录/噪声）与自适应树深度。
  - `docstrings.py`：**数据层唯一做文件 I/O 的模块**，从源码提取文档字符串。
  - `project_info.py`：项目元数据（README/配置文件/顶层 docstring）。
  - `ref_docs.py`：参考文档发现（环境变量驱动）。
  - `hashes.py`：基于结构数据的缓存失效哈希（非文本内容）。
- **门面层**（`__init__.py`）：聚合对外导出，`__all__` 列出全部公共符号。

## 文件清单

- `__init__.py` — 数据层门面，聚合导出全部公共符号（见下方 `__all__`）。
- `db.py` — CodeGraph SQLite 只读访问边界；定义 `CodeGraphDB` 及 `FileRow`/`NodeRow`/`EdgeRow` 三个冻结数据类。
- `modules.py` — 文件级依赖提取；将符号级 `imports`/`calls` 边映射为 `Module`/`ModuleEdge`，含外部依赖（`external::` 前缀）。
- `aggregate.py` — 文件边聚合到目录视图（`directory_dependencies`/`directory_edges`/`directory_symbols`）。
- `architecture.py` — 语言无关架构特征：目录中心性、拓扑分层、环检测、跨目录公共 API、外部依赖聚合。
- `files.py` — 文件列表与层级目录树，合并 `.gitignore` 与 `.codesense/.codesenseignore` 过滤。
- `structure.py` — 顶层目录分类（主/辅助/噪声）与自适应最大深度计算。
- `docstrings.py` — 多语言文档字符串提取（Python/TS/JS/Go/Rust/Erlang/Ruby/Shell），数据层唯一文件 I/O 模块。
- `project_info.py` — 项目身份信息提取（README/配置文件/顶层包 docstring）与技术栈提示。
- `ref_docs.py` — 参考文档目录发现（`CODESENSE_REF_DOCS_DIR` 环境变量驱动）。
- `hashes.py` — project_map 段缓存失效哈希（基于结构数据，非文本内容）。

## 对外接口

### 核心类

- `CodeGraphDB(project_root: str | Path)` — SQLite 只读封装，上下文管理器。方法：`iter_files() -> Iterator[FileRow]`、`iter_nodes(kinds=None) -> Iterator[NodeRow]`、`iter_edges(kinds=None) -> Iterator[EdgeRow]`、`get_node(node_id) -> NodeRow | None`、`stats() -> dict[str, object]`、`close()`。
- `FileRow`（frozen dataclass）— `path: str, language: str, size: int, node_count: int`。
- `NodeRow`（frozen dataclass）— `id, kind, name, qualified_name, file_path, language, start_line, end_line, signature: str | None`。
- `EdgeRow`（frozen dataclass）— `source, target, kind, line: int | None`。
- `Module`（frozen dataclass）— `id, file_path, language, package_id`（id = POSIX 文件路径）。
- `ModuleEdge`（frozen dataclass）— `source, target, kind("imports"|"calls"), is_external: bool`。
- `DirCentrality`（frozen dataclass）— `directory, fan_in, fan_out, fan_out_external: int`。
- `ArchitectureFeatures`（frozen dataclass）— `centrality: dict[str, DirCentrality], layers: list[list[str]], cycles: list[list[str]], public_api: dict[str, list[str]], external_by_dir: dict[str, list[str]]`。
- `DirectoryNode`（dataclass）— `name, path, files: list[FileRow], subdirs: dict[str, DirectoryNode]`；方法 `to_dict() -> dict`。
- `TopLevelDir`（frozen dataclass）— `name, file_count: int, is_auxiliary: bool, category: str | None`。
- `IdentitySource`（frozen dataclass）— `kind, path, content: str`。

### 核心函数

**模块依赖（modules.py）**
- `list_modules(db: CodeGraphDB) -> list[Module]` — 返回所有项目文件为 Module。
- `module_dependencies(db, *, include_external=True, include_calls=True, include_imports=True) -> list[ModuleEdge]` — 计算文件级依赖边集。
- `to_file_dependency_dict(edges) -> dict[str, dict[str, list[str]]]` — 文件级视图 `{file: {imports, calls}}`，外部目标加 `external::` 前缀。
- `to_package_dependency_dict(edges, modules, *, include_self_loops=False) -> dict[str, dict[str, list[str]]]` — 包/目录级聚合视图。

**目录聚合（aggregate.py）**
- `directory_dependencies(edges, modules, *, max_depth=None, include_external=True, include_self_loops=False) -> dict[str, dict[str, list[str]]]` — 按 (源目录, 目标目录) 分组。
- `directory_edges(edges, modules, *, ...) -> list[tuple[str, str, str]]` — 扁平 `(src, tgt, kind)` 列表。
- `directory_symbols(db, *, max_depth=None, kinds=("function","class","method"), max_per_dir=None) -> dict[str, list[dict[str, str]]]` — 目录→符号列表。

**架构特征（architecture.py）**
- `compute_centrality(edges, modules, *, max_depth=None) -> dict[str, DirCentrality]` — 目录 fan-in/fan-out/外部 fan-out。
- `find_cycles(edges, modules, *, max_depth=None) -> list[list[str]]` — SCC（size>1）即真实环。
- `topological_layers(edges, modules, *, max_depth=None) -> list[list[str]]` — 目录分层（layer 0 = 基础），环收缩为超节点。
- `cross_dir_public_api(db, *, max_depth=None, max_per_dir=30, symbol_kinds=("function","class","method","variable")) -> dict[str, list[str]]` — 被目录外代码 import 的符号。
- `external_dependencies_by_dir(edges, modules, *, max_depth=None, max_per_dir=20) -> dict[str, list[str]]` — 每目录外部依赖聚合。
- `architecture_features(db, edges, modules, *, max_depth=None) -> ArchitectureFeatures` — 一次性计算全部架构信号。

**文件/结构（files.py / structure.py）**
- `list_files(db) -> list[FileRow]` — 扁平文件列表（排除 gitignored）。
- `directory_tree(db) -> DirectoryNode` — 层级目录树。
- `classify_top_dirs(all_file_paths: list[str]) -> list[TopLevelDir]` — 顶层目录分类（按文件数降序）。
- `auxiliary_category(name: str) -> str | None` — 辅助目录类别标签。
- `compute_tree_max_depth(file_paths, aux_dir_names=None, floor=3) -> int` — 自适应最大深度。

**文档字符串（docstrings.py）**
- `extract_file_docstring(file_path, language) -> str | None` — 文件级 docstring。
- `extract_symbol_docstrings(file_path, language, nodes: list[NodeRow]) -> dict[str, str]` — `{node.id: docstring}`。
- `is_enabled() -> bool` — 是否启用提取（`CODESENSE_EXTRACT_DOCSTRINGS` 控制）。

**项目信息（project_info.py）**
- `collect_identity_sources(project_root, db) -> list[IdentitySource]` — 按优先级收集身份源（README > 配置 > docstring）。
- `read_readme(project_root) -> IdentitySource | None`。
- `extract_tech_stack_hint(sources) -> dict[str, str]` — 技术栈提示。

**参考文档（ref_docs.py）**
- `discover_ref_docs(project_root) -> list[Path]` — 发现参考文档路径。
- `ref_docs_prompt_section(project_root) -> str` — 生成提示词段落（无文档返回 `""`）。

**哈希（hashes.py）**
- `compute_identity_hash(sources) -> str`、`compute_structure_hash(top_dirs) -> str`、`compute_architecture_hash(module_dir_groups) -> str`、`compute_dependencies_hash(edges) -> str` — 各段缓存失效哈希。

### 常量
- `EXTERNAL_PREFIX = "external::"`（modules.py）— 外部依赖目标前缀。
- `AUXILIARY_DIR_NAMES` / `AUXILIARY_CATEGORY`（structure.py）— 辅助目录名集合与类别映射。
- `DB_RELATIVE_PATH = .codegraph/codegraph.db`（db.py）。

## 上下游关系

- **上游（依赖此模块的目录）**：`src/codesense_v1/summarizer`、`src/codesense_v1/tools`、`scripts`、`tests`。
- **下游（此模块依赖的目录）**：无。数据层位于第 0 层（基础层），不依赖项目内其他模块，仅依赖标准库与 `pathspec`。

## 外部依赖

标准库：`collections`、`collections.abc`、`dataclasses`、`hashlib`、`json`、`os`、`pathlib`、`re`、`sqlite3`、`sys`、`typing`。
第三方：`pathspec`（gitignore 风格路径匹配，用于 `files.py`）。

## 实现约束清单

1. **只读边界**：`CodeGraphDB` 以 `mode=ro` URI 打开 SQLite，绝不写入；schema 变更只改 `db.py`。
2. **唯一文件 I/O 例外**：`docstrings.py` 是数据层唯一读取源码文件的模块，其余模块只读 SQLite；可通过 `CODESENSE_EXTRACT_DOCSTRINGS=false` 全局禁用。
3. **外部依赖标识**：外部 import 目标在 dict 视图中统一加 `external::` 前缀，内部/外部视觉无歧义。
4. **calls 边可信度过滤**：`module_dependencies` 仅信任 target 为可调用符号（function/method/class）的 calls 边；跨文件 calls 边需已有对应 imports 边才采纳，避免 CodeGraph 误解析同名方法产生伪边。
5. **自环剔除**：`module_dependencies` 默认忽略文件依赖自身的边；目录聚合函数默认 `include_self_loops=False`。
6. **哈希基于结构而非文本**：`hashes.py` 各哈希基于文件清单/目录列表/边集合等结构数据，Agent 改写措辞不会触发缓存失效。
7. **图算法迭代实现**：`_tarjan_sccs` 与 `topological_layers` 用迭代 DFS 避免大图递归栈溢出，并动态抬高 `sys.setrecursionlimit`。
8. **语言无关架构特征**：`architecture.py` 全部为纯图度量，不查阅任何语言特定导出规则（`__all__`/`pub`/大写名），跨语言一致。
9. **best-effort 降级**：`docstrings.py` 所有公共函数在无法定位时返回 `None`/空 dict，调用方须优雅降级。
10. **路径规范化**：文件 id 统一为 POSIX 路径（`\` → `/`）；Python resolve_id 为点号模块名，TS/JS 为去扩展名路径。
