# 数据层

## 概述
封装 CodeGraph SQLite 知识图谱的只读查询，提供文件/模块/依赖/架构特征与 docstring 提取能力，是整个项目的数据供给基础。

## 对外接口

### DB 访问（db.py）
- CodeGraphDB(project_root: str | Path)：只读封装 .codegraph/codegraph.db，支持上下文管理。
  - iter_files() -> Iterator[FileRow]：迭代所有索引文件。
  - iter_nodes(kinds=None) -> Iterator[NodeRow]：迭代符号节点（可按 kind 过滤）。
  - iter_edges(kinds=None) -> Iterator[EdgeRow]：迭代依赖边。
  - get_node(node_id: str) -> NodeRow | None：按 id 查节点。
  - stats() -> dict[str, object]：返回 DB 统计信息。
  - close() -> None：关闭连接。
- 数据行类：FileRow、NodeRow、EdgeRow。

### 文件视图（files.py）
- list_files(db: CodeGraphDB) -> list[FileRow]：返回所有索引文件的扁平列表。
- directory_tree(db: CodeGraphDB) -> DirectoryNode：构建索引文件的层级目录树（根节点路径为空）。
- DirectoryNode：目录树节点类，含 to_dict() -> dict[str, object]。

### 模块依赖（modules.py）
- list_modules(db: CodeGraphDB) -> list[Module]：返回所有项目文件作为 Module（id = POSIX 路径）。
- module_dependencies(db, *, include_external=True, include_calls=True, include_imports=True) -> list[ModuleEdge]：计算文件级依赖边集合。
- to_file_dependency_dict(edges) -> dict[str, dict[str, list[str]]]：文件级视图 {file: {imports, calls}}。
- to_package_dependency_dict(edges, modules, *, include_self_loops=False) -> dict[str, dict[str, list[str]]]：包/目录级聚合视图。
- 数据类：Module（单个源文件）、ModuleEdge（有向依赖边，含 file→external）。

### 目录聚合（aggregate.py）
- directory_dependencies(edges, modules, *, max_depth=None, include_external=True, include_self_loops=False) -> dict[str, dict[str, list[str]]]：按 (src_dir, tgt_dir) 分组边。
- directory_edges(edges, modules, *, ...) -> list[tuple[str, str, str]]：扁平化版本，返回 (src, tgt, kind) 列表。
- directory_symbols(db, *, max_depth=None, kinds=(function,class,method), max_per_dir=None) -> dict[str, list[dict[str, str]]]：目录→符号列表映射。

### 架构特征（architecture.py）
- compute_centrality(edges, modules, *, max_depth=None) -> dict[str, DirCentrality]：每目录 fan-in/fan-out（内部）与外部 fan-out 计数。
- find_cycles(edges, modules, *, max_depth=None) -> list[list[str]]：返回大小 >1 的强连通分量（真实环）。
- topological_layers(edges, modules, *, max_depth=None) -> list[list[str]]：从基础层（layer 0）向上分层。
- cross_dir_public_api(db, *, max_depth=None, max_per_dir=30, symbol_kinds=(function,class,method,variable)) -> dict[str, list[str]]：每目录被外部 import 的符号列表。
- external_dependencies_by_dir(edges, modules, *, max_depth=None, max_per_dir=20) -> dict[str, list[str]]：每目录的外部（external::）import 目标聚合。
- architecture_features(db, edges, modules, *, max_depth=None) -> ArchitectureFeatures：一次性计算所有架构信号。
- 数据类：DirCentrality（目录连通性指标）、ArchitectureFeatures（架构信号束）。

### Docstring 提取（docstrings.py）
- is_enabled() -> bool：除非 CODESENSE_EXTRACT_DOCSTRINGS=false 否则返回 True。
- extract_file_docstring(file_path, language) -> str | None：提取模块/文件级 docstring。
- extract_symbol_docstrings(file_path, language, nodes: list[NodeRow]) -> dict[str, str]：返回 {node.id: docstring}。
- 内部多语言提取器：_python_triple_quote、_jsdoc_backward、_jsdoc_forward、_line_comment_backward、_line_comment_forward、_file_docstring_python、_file_docstring_for_lang、_symbol_docstring_for_lang。

## 内部文件

- src/codesense_v1/data/__init__.py：包初始化，导出数据层公共接口。
- src/codesense_v1/data/db.py：只读访问 .codegraph/codegraph.db，定义 CodeGraphDB 与 FileRow/NodeRow/EdgeRow 行类。
- src/codesense_v1/data/files.py：从 CodeGraph 索引构建文件列表与目录树。
- src/codesense_v1/data/modules.py：文件级依赖边提取与模块/包级依赖聚合。
- src/codesense_v1/data/aggregate.py：将文件级边聚合为目录级视图（依赖、符号）。
- src/codesense_v1/data/architecture.py：从依赖图派生语言无关的架构特征（中心性、环、拓扑层、跨目录公共 API、外部依赖）。
- src/codesense_v1/data/docstrings.py：从源文件提取文件级与符号级 docstring，支持 Python/JS/TS 等多语言。

## 依赖关系

### 上游依赖
无。数据层位于基础层（拓扑第 0 层），不依赖项目内其他模块。

### 下游依赖（依赖本模块的目录）
- src/codesense_v1/summarizer：摘要生成层调用数据层查询 DB 并计算架构特征。
- src/codesense_v1/tools：MCP 工具层调用数据层响应查询请求。
- scripts：辅助脚本直接使用数据层。
- tests：测试代码覆盖数据层查询与聚合逻辑。

## 外部依赖

仅使用 Python 标准库：
- sqlite3：访问 CodeGraph SQLite 数据库。
- dataclasses：定义 FileRow/NodeRow/EdgeRow/Module/ModuleEdge/DirCentrality/ArchitectureFeatures/DirectoryNode 等数据类。
- collections.abc：类型注解（Iterable/Iterator）。
- pathlib：跨平台路径操作。
- os：路径与环境变量（CODESENSE_EXTRACT_DOCSTRINGS）。
- re：docstring 与 import 解析的正则匹配。
- sys：辅助用途。
