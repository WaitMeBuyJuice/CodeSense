# 缓存层模块

## 概述
管理 `.codesense/` 目录的缓存读写与失效校验，提供项目级与模块级两层缓存，基于 DB SHA-256 哈希与模块内容哈希做失效判定。

## 对外接口

通过 `__init__.py` re-export 13 个公开函数（私有 `_` 前缀不导出）：

### 哈希与校验
- `db_hash(db_path: Path) -> str` — 计算 CodeGraph SQLite 文件 SHA-256，DB 缺失抛 `FileNotFoundError`
- `is_cache_valid(codesense_dir: Path, current_hash: str) -> bool` — 比对 `meta.json` 中 `db_hash`，任何异常返回 `False`

### 项目级缓存
- `read_project_map(codesense_dir: Path) -> str | None` — 读 `project_map.md`，失败返回 `None`
- `write_project_map(codesense_dir: Path, content: str, current_hash: str) -> None` — 写 `project_map.md` + 更新 `meta.json`，自动建目录

### 模块索引缓存
- `read_modules_index(codesense_dir: Path) -> dict[str, object] | None` — 读 `modules_index.json`（含 `modules` 与 `auxiliary_dirs`）
- `write_modules_index(codesense_dir, modules, current_hash, aux_dirs=None) -> None` — 写索引，调用 `_prune_stale_modules` 清理已移除模块的 `.md` 与哈希条目，保留幸存模块摘要

### 模块级缓存
- `read_module(codesense_dir: Path, module_key_str: str) -> str | None` — 读 `modules/<key>.md`
- `write_module(codesense_dir, module_key_str, module_name, summary, current_hash, module_content_hash="") -> None` — 写模块摘要 + 可选写模块哈希 + 更新 `meta.json`；`module_name` 参数保留仅为 API 兼容（`noqa: ARG001`）
- `read_module_hashes(codesense_dir: Path) -> dict[str, str]` — 读 `modules/.hashes.json`，兼容旧扁平格式与新版 `{hash, generated_at}` 格式
- `write_module_hash(codesense_dir, module_key_str, module_hash) -> None` — upsert 模块哈希 + 时间戳

### 失效与键生成
- `invalidate(codesense_dir: Path) -> None` — 全量清空（project_map / modules_index / meta / modules/ 全部），静默忽略缺失
- `module_key(module_path: str) -> str` — 路径转文件名键（`/` `\` → `_`）
- `safe_key(module_name: str) -> str` — 模块名转安全文件名，替换非法字符 `[\/:*?"<>|\x00-\x1f]`，截断 100 字符，空串兜底为 `_`

## 内部文件

- `__init__.py` — re-export 13 个公开函数，定义 `__all__`
- `cache.py` — 全部实现，含 5 个私有 helper（`_now_iso` / `_meta_path` / `_write_meta` / `_clear_modules_dir` / `_prune_stale_modules`）

## 缓存目录结构

```
.codesense/
├── meta.json              # {db_hash, generated_at}
├── project_map.md         # 项目架构概览
├── modules_index.json     # {generated_at, modules[], auxiliary_dirs[]}
└── modules/
    ├── .hashes.json       # {module_key: {hash, generated_at}}
    └── <module_key>.md    # 各模块摘要
```

## 设计要点

1. **两层失效策略**：项目级用 DB 文件 SHA-256（`meta.json.db_hash`），模块级用模块内容哈希（`modules/.hashes.json`），粒度独立。
2. **容错约定**：所有 `read_*` 失败返回 `None`/`{}`（视为 cache miss），`write_*` 传播 `OSError`，`invalidate` 静默忽略缺失——调用方无需 try/except 读路径。
3. **增量清理**：`write_modules_index` 不全量清空，仅 `_prune_stale_modules` 移除不再存在的模块文件与哈希，幸存模块摘要保留由 per-module 哈希决定是否重生。
4. **跨平台文件名安全**：`safe_key` 同时处理 Windows + Linux 非法字符，截断 100 字符，空串兜底 `_`；原模块名存 JSON payload 而非文件名。
5. **哈希格式向后兼容**：`read_module_hashes` 同时识别旧扁平 `{key: hash_str}` 与新版 `{key: {hash, generated_at}}`。
6. **流式哈希**：`db_hash` 按 64KB chunk 读取，避免大 DB 一次性载入内存。
