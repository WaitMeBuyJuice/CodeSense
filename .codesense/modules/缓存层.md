# 缓存层

## 概述
管理 `.codesense/` 目录下缓存文件（项目架构图、模块索引、模块摘要）的读写、校验与失效，基于 DB 哈希判断缓存是否过期。

## 对外接口

### 缓存校验
- `db_hash(db_path: Path) -> str`：计算 CodeGraph DB 文件的 SHA-256 哈希，作为缓存版本标识。
- `is_cache_valid(codesense_dir: Path, current_hash: str) -> bool`：判断缓存是否有效——`meta.json` 存在且其 `db_hash` 与当前哈希一致。
- `invalidate(codesense_dir: Path) -> None`：删除 `project_map.md`、`modules_index.json`、`meta.json` 及整个 `modules/` 目录，强制下次重建。

### 项目架构图缓存
- `read_project_map(codesense_dir: Path) -> str | None`：读取 `project_map.md`，缺失或不可读返回 `None`。
- `write_project_map(codesense_dir: Path, content: str, current_hash: str) -> None`：写入 `project_map.md` 并更新 `meta.json` 中的哈希。

### 模块索引缓存
- `read_modules_index(codesense_dir: Path) -> dict[str, object] | None`：解析 `modules_index.json`，缺失返回 `None`。
- `write_modules_index(codesense_dir: Path, modules: list[dict[str, object]], current_hash: str, aux_dirs: list[dict[str, object]] | None = None) -> None`：写入索引、清理失效模块文件、更新 `meta.json`。

### 单模块摘要缓存
- `read_module_hashes(codesense_dir: Path) -> dict[str, str]`：读取 `modules/.hashes.json` 中的每模块哈希表。
- `write_module_hash(codesense_dir: Path, module_key_str: str, module_hash: str) -> None`：更新指定模块的哈希与 `generated_at` 时间戳。
- `read_module(codesense_dir: Path, module_key_str: str) -> str | None`：读取 `modules/<module_key>.md` 模块摘要。
- `write_module(codesense_dir: Path, module_key_str: str, module_name: str, summary: str, current_hash: str, module_content_hash: str = "") -> None`：写入模块摘要、更新模块哈希与 `meta.json`。

### 文件名键生成
- `module_key(module_path: str) -> str`：将模块路径转换为安全的文件名键。
- `safe_key(module_name: str) -> str`：由模块名生成人类可读的文件名键。

## 内部文件

- `src/codesense_v1/cache/__init__.py`：包初始化，导出缓存层公共接口。
- `src/codesense_v1/cache/cache.py`：缓存层全部实现——读写 `project_map.md`、`modules_index.json`、`modules/<key>.md`、`meta.json`、`modules/.hashes.json`，并提供哈希校验、失效清理与陈旧模块裁剪逻辑。

## 依赖关系

### 上游依赖
无。缓存层位于基础层（拓扑第 0 层），不依赖项目内其他模块。

### 下游依赖（依赖本模块的目录）
- `src/codesense_v1/summarizer`：摘要生成层调用缓存读写接口协调数据层与缓存。
- `src/codesense_v1/tools`：MCP 工具层直接读写缓存以响应 `project_map` / `explore_module` 请求。
- `tests`：测试代码覆盖缓存读写与失效行为。

## 外部依赖

仅使用 Python 标准库：
- `datetime`：生成 ISO 格式时间戳（`generated_at`）。
- `hashlib`：计算 DB 文件 SHA-256 哈希。
- `json`：序列化/反序列化 `meta.json`、`modules_index.json`、`.hashes.json`。
- `pathlib`：跨平台路径操作。
- `re`：文件名键的安全化处理。
