# 任务 3：codegraph_explore 子系统协作分析（codegraph-main，中等，纯问答）

## 一、实验内容

**任务类型**：跨子系统协作问答（无代码改动，测协作链路理解）

**任务描述**：
> 在 codegraph 项目中，当 Agent 调用 `codegraph_explore` MCP 工具时，从接收 MCP 请求到返回结果，涉及哪些核心子系统？请按调用顺序列出，并说明每个子系统在这次调用中的角色。

**涉及项目**：`E:\Python_Project\codegraph-main`

**为什么选这个任务**：
- 不在提示词出现"用 CodeSense"暗示
- "涉及哪些核心子系统" 强烈命中 `project_map` description
- TypeScript 陌生项目，无法靠训练集记忆
- 中等复杂度：核心调用链涉及 4-5 个子系统
- 纯问答无代码改动，**实验后无需重置代码库**

**Ground Truth（基于 README + `src/` 实际结构）**：

| # | 子系统 | 角色 |
|---|-------|------|
| 1 | `mcp/` | 接收 MCP `tools/call` 请求，解析参数，分发到 explore handler |
| 2 | `graph/` 或图查询层 | 执行图遍历：从查询关键词找入口节点、扩展邻域、计算 blast radius |
| 3 | `db/` 或 DatabaseConnection | SQLite 查询：FTS5 全文搜索、节点查询、边查询 |
| 4 | `extraction/`（可选提及） | 已有的索引数据来源（提供 nodes/edges） |
| 5 | `resolution/`（可选提及） | 提供符号引用解析数据 |

**Ground Truth 关键概念清单**（共 8 项）：
1. MCP server 入口（`src/mcp/`）
2. 工具调用分发机制
3. FTS5 全文搜索（找入口符号）
4. 图遍历（callers / callees / 邻域）
5. SQLite 数据库查询
6. 节点 + 边数据来源（extraction 索引产物）
7. 返回结果包含：符号源代码 + 关系图 + blast radius
8. 整个流程"近零文件读"（README 强调的特性）

**验收**：无自动验收，人工对照计算命中率。

**实验次数**：每组 3 次（关键任务）
