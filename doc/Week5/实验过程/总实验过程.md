## 三、实验前准备

### 3.1 代码库状态确认

```bat
cd /d E:\Python_Project\CodeSense_V1
git status                      ← 必须 clean

cd /d E:\Python_Project\codegraph-main
git status                      ← 同上
```

### 3.2 CodeSense 安装

```bat
cd /d E:\Python_Project\CodeSense_V1
uv tool install --editable . --reinstall
where codesense_v1
```

### 3.3 CodeGraph 索引确认

```bat
dir E:\Python_Project\CodeSense_V1\.codegraph\codegraph.db
dir E:\Python_Project\codegraph-main\.codegraph\codegraph.db
```

若不存在，在对应目录执行 `codegraph init`。

### 3.4 CodeSense modules_index 确认

两个项目各自需要预先生成 `.codesense/modules_index.json`：
- 切 `CODESENSE_PROJECT_ROOT` 到目标项目 → 重装 → 重启 VSCode → 读取一次 `codesense://project_map` 资源
- 检查文件生成：`dir <project>\.codesense\modules_index.json`

## 五、单次实验执行流程

```
① 确认 MCP 配置已切换 + VSCode 已重启
② 确认代码库状态干净（git status clean）—— 任务1 必查，任务2/3/4 可跳过
③ 新开一个 CodeMaker 对话
④ 把 [Phase 1 提示词] 粘贴给 Agent → Agent 执行任务
⑤ Agent 输出实验报告 → 立刻从 CodeMaker UI 抄 Token + 耗时
⑥ 把 [Phase 2 提示词] 粘贴给 Agent（同一会话）→ Agent 输出 GT 对照表
⑦ 把两份报告摘抄到对应任务的结果表
⑧ 执行代码库重置（仅任务1 需要）
⑨ 进入下一组实验或下一任务
```

## 六、代码库重置（仅任务 1 需要）

任务 1 实际改 CodeSense_V1 代码，必须在下次实验前重置：

```bat
cd /d E:\Python_Project\CodeSense_V1
git checkout -- .               ← 丢弃所有未提交改动
git status                      ← 确认 clean
uv run pytest -q                ← 确认基线 164 passed
```

任务 2、3、4 是纯问答或评估（不实际改代码），**无需重置**。

## 七、推荐执行顺序

按"同一组连续做完所有任务，再切到下一组"的顺序，减少 MCP 配置切换次数：

```
组1（关 CG + 关 CS + 关 Skill）
  ├─ 任务1（3 次，每次后 git checkout）
  ├─ 任务2（3 次）
  ├─ 任务3（3 次，先切 CODESENSE_PROJECT_ROOT 到 codegraph-main，但组1 CS 关着所以无所谓）
  └─ 任务4（3 次）

组2（开 CG + 关 CS + 关 Skill）
  ├─ 任务1（3 次）
  ├─ 任务2（3 次）
  ├─ 任务3（3 次）
  └─ 任务4（3 次）

组3（开 CG + 开 CS + 关 Skill）
  ├─ 任务1（3 次，CODESENSE_PROJECT_ROOT = CodeSense_V1）
  ├─ 任务2（3 次，同上）
  ├─ ⚠️ 切 CODESENSE_PROJECT_ROOT = codegraph-main，重启 VSCode
  ├─ 任务3（3 次）
  └─ 任务4（3 次）

组4（开 CG + 开 CS + 开 Skill）
  └─ 同组3 顺序，额外激活 codesense-workflow Skill
```
