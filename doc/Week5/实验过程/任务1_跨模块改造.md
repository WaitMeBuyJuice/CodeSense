# 任务 1：跨模块缓存策略改造（CodeSense_V1，复杂）

## 一、实验内容

**任务类型**：跨模块代码改造（有 ground truth，测找全率 + 误改率 + 测试通过）

**任务描述**：
> 把当前 `cache` 模块的"全量失效"策略改造成"按 hash 增量失效"——hash 变化时不再清空所有缓存，而是分别判断 project_map 和每个模块的 hash，只清除真正变化的部分，保留未变化的缓存内容。改完跑测试确认通过。

**涉及项目**：`E:\Python_Project\CodeSense_V1`

**为什么选这个任务**：
- 不在提示词里出现"用 CodeSense / explore_module"等暗示
- "理解当前策略 → 改造" 自然触发 `explore_module(cache)` 和 `project_map`（命中 description 的"询问某模块的作用或策略"+"改动某模块前需先了解其结构和接口契约"）
- 跨 5 个文件，需要发现 `summarizer` 对 cache 的隐含依赖（**反假饱和关键观察点**）

**Ground Truth（已 grep 验证）**：

核心必改文件（共 3 个）——找全率分母：

| # | 文件 | 改动原因 |
|---|------|---------|
| 1 | `src/codesense_v1/cache/cache.py` | 核心改造：`invalidate()` + `is_cache_valid()` 改为按 hash 选择性清除（当前 meta.json 仅存单一全局 db_hash） |
| 2 | `src/codesense_v1/summarizer/summarizer.py` | 唯一调用 `cache.invalidate()` 的源文件（第 41、87 行），需相应调整 |
| 3 | `tests/test_cache.py` | 现有失效测试需调整，新增增量失效测试 |

条件性改动文件（共 2 个）——加分项，不算分母：

| # | 文件 | 改动原因 |
|---|------|---------|
| 4 | `src/codesense_v1/cache/__init__.py` | 若新增辅助函数（如 `invalidate_partial`）才需 re-export |
| 5 | `tests/test_summarizer.py` | 若 fixture 依赖完整 invalidate 行为才需调整 |

**不该改**：上述清单以外的所有源文件

**验收命令**：`uv run pytest -q`（应全部通过）

**实验次数**：每组 3 次（关键任务）
