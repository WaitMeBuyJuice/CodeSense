## 一、本次完成的工作

### 背景

原有 14 个 MCP 工具中，7 个 `get_*_prompt` 工具是纯 prompt 工厂，Agent 调用它们本身不产生任何决策价值，仅是"取 prompt → 推理 → save"流程中的中间步骤。

### 方案（方向 A：prompt 嵌入）

将 prompt 逻辑直接内嵌至对应的 explore / project_map 工具：

- `explore_module` cache miss → 直接调 `summarizer.get_module_prompt()` 拿 prompt，嵌入返回体；保留子 Agent（推荐）/ 主 Agent 两种执行方式说明
- `explore_submodule` cache miss → 直接调 `summarizer.get_submodule_prompt()` 嵌入返回，同上
- `project_map` 缺段 → 各段 prompt 直接附在对应步骤块内；依赖 `03_modules` 未完成的段跳过 prompt 并标注阻塞原因

### 工具数变化

**14 → 7**（删除 7 个工具）

**删除的工具（文件已删除）**：

| 工具 | 原职责 |
|------|--------|
| `get_identity_segment_prompt` | 返回 01_identity prompt |
| `get_modules_segment_prompt` | 返回 03_modules prompt |
| `get_constraints_segment_prompt` | 返回 04_constraints prompt |
| `get_flows_segment_prompt` | 返回 05_flows prompt |
| `get_concepts_segment_prompt` | 返回 06_concepts prompt |
| `get_module_prompt` | 返回模块总览 prompt |
| `get_submodule_prompt` | 返回子模块文档 prompt |

**保留的工具（7 个）**：

| 工具 | 用途 |
|------|------|
| `project_map` | 项目概览（缺段时内嵌各段 prompt） |
| `explore_module` | 模块总览（miss 时内嵌 prompt） |
| `explore_submodule` | 子模块文档（miss 时内嵌 prompt） |
| `save_project_map_segment` | 保存 01/03/04/05/06 段 |
| `save_module_summary` | 保存模块总览 |
| `save_submodule_summary` | 保存子模块文档 |
| `submit_project_map` | 提交模块划分（驱动 modules_index 更新） |
