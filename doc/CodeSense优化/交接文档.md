## 一、项目简介

CodeSense 是一个 **MCP（Model Context Protocol）服务**，通过读取 CodeGraph 代码图谱，为 AI Agent 自动生成项目架构概览与模块摘要。采用 **Lazy Cache with Agent-Driven Generation** 模式，知识文档按需生成、增量更新。

- 语言：Python 3.14+
- 核心依赖：mcp、openai、jsonschema、pathspec
- 数据来源：CodeGraph（`.codegraph/codegraph.db`）
- 缓存目录：`.codesense/`

## 二、当前代码状态（本轮优化后）

### 2.1 已完成的核心变更

**project_map 4 段分片架构（本轮最大改动）**

`project_map.md` 拆分为 4 个独立 segment，分段存储于 `.codesense/project_map_segments/`：

| Segment | 内容 | 生成方式 | 失效 Hash 依据 |
|---------|------|---------|--------------|
| `01_identity.md` | 仓库定位 + 技术栈 | LLM（`get_identity_segment_prompt`） | README + pyproject.toml 等 |
| `02_structure.md` | 顶层目录树 | 程序化（`render_structure_segment`） | 一级目录列表 |
| `03_modules.md` | 模块列表 | LLM（`get_modules_segment_prompt` → `submit_project_map`） | 模块目录分配 |
| `04_dependencies.md` | 依赖图 + 上下游表 | 程序化（`render_dependencies_segment`） | 模块间导入边集 |

四段通过 `cache.render_project_map()` 合并写入最终 `project_map.md`。

| `get_modules_segment_prompt` | 返回 03_modules 的 LLM prompt（原 get_project_map_prompt） |

## 四、关键设计说明

### 4.1 Lazy Cache + Agent-Driven Generation

所有知识文档都不预生成，只在 Agent 调用 `project_map` / `explore_module` 时检查缓存：
- **缓存命中** → 直接返回
- **缓存未命中** → 返回引导文本，Agent 调用 prompt 工具 → 生成内容 → 写缓存 → 重调获取

### 4.2 segments 的失效逻辑

**注意**：`04_dependencies` 在首次 `project_map` 调用时会先用空模块生成（路径级别），等 `submit_project_map` 完成后会**强制重写**（带模块名）。这是设计行为，不是 bug。

**测试失效**：
- `01_identity`：改 pyproject.toml → `codegraph sync` → 调 `project_map`
- `02_structure`：增删文件 → `codegraph sync` → 调 `project_map`（自动重生成）
- `04_dependencies`：增删 import → `codegraph sync` → 调 `project_map`（自动重生成）
- `03_modules`：重新提交模块划分（`submit_project_map`）

### 4.3 单文件模块的处理

当模块是 `errors.py` 这种顶层独立文件时，LLM 可在模块划分时提交文件路径：
```
错误定义|...|src/codesense_v1/errors.py
```
系统会正确识别并存储为 `files` 字段，在依赖图和模块列表中显示正确路径。
