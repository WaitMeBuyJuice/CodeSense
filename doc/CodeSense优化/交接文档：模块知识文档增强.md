## 二、本次（前一段对话）完成的工作

### 2.1 P0.3：模块总览 prompt 重构

`_build_module_prompt` 输出格式改为严格 5 章节，禁止 LLM 自增「对外接口」等额外章节：

1. **一句话定位**（10-20 字）
2. **架构简析**（模块内部分层）
3. **文件清单**（每文件一行，`__init__.py` 导出信息在此列出）
4. **上下游关系**（数据来自 imports 边，置信度 extracted）
5. **实现约束清单**（只写模块内部约束，不重复 04_constraints 的跨模块禁忌）

**去除原因**：对外接口（函数签名）由 CodeGraph 实时提供，文档中冗余；关键常量字面量生成慢、失效频繁，性价比低。

### 2.2 P0.4：子模块文档 + 3 个 MCP 工具

新增 `explore_submodule` / `get_submodule_prompt` / `save_submodule_summary`，与现有 `explore_module` 完全对称。

子模块文档固定 4 章节：文件概述 / 对外接口 / 跨模块依赖 / 典型调用链。

特殊规则：
- `__init__.py` 不生成独立子模块文档，导出信息合并到模块总览文件清单
- 单文件模块（除 `__init__.py` 外 ≤1 个 .py 文件）跳过子模块文档，`explore_submodule` 返回引导信息

### 2.3 存储结构重构

| 维度 | 变更前 | 变更后 |
|---|---|---|
| 模块总览路径 | `modules/<module>.md` | `modules/<module>/<module>_overview.md` |
| 子模块路径 | `modules/<module>/<file_key>.md` | `modules/<module>/<module>_<basename>.md` |
| 总览 hash 存储 | `modules/.hashes.json` (key=模块名) | `modules/<module>/.hashes.json` (key="overview") |
| 子模块 hash 存储 | `modules/<module>/.hashes.json` (key=文件名) | 同上 (key=`<module>_<basename>`) |
| 模块名 | LLM 自由生成（可能中文）| 强制英文 snake_case |
