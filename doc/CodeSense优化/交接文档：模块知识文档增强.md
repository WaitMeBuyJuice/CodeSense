### 3.3 实现前需讨论的关键点

新对话开始时建议与用户讨论：

1. **存储结构**：子模块文档是 flat（`modules/<module>__<file>.md`）还是 nested（`modules/<module>/<file>.md`）？
2. **`__init__.py` 处理**：是否单独生成子模块文档，还是合并到模块文档？
3. **单文件模块特例**：如 `errors.py` 这种已经是单文件的模块，是否再生成子模块文档？
4. **失效粒度**：模块 hash 和子模块 hash 的关系（修改一个文件只失效对应子模块？还是连带失效整个模块？）
5. **新增工具**：是否新增 `explore_submodule` / `get_submodule_prompt` / `save_submodule_summary` 三个工具，还是扩展现有工具？
6. **模块文档增强**：除了 P0.3 列出的几项，是否参考 CodeMap.md 价值分析加入「实现约束清单」「关键常量字面值」「retrieval_hints」？
