# B-3: 创建测试包占位 tests/__init__.py

## 任务背景

`pyproject.toml` 中 `[tool.pytest.ini_options] testpaths = ["tests"]`、`pythonpath = ["src"]`。`tests/` 目录需要存在并被 pytest 收集。

---

## 实现目标

创建空的 `tests/__init__.py` 占位，使 `tests` 成为可被发现的包目录，并能让 `uv run pytest tests --collect-only` 正常返回（即使尚无测试用例）。

---

## 需要实现的文件

- `tests/__init__.py`

不得创建/修改任何其他文件。

---

## 内容契约

- 空文件（0 字节）即可

---

## 验收标准

- 文件存在
- `uv run pytest tests --collect-only` 退出码 0 或 5（`no tests collected` 也算通过）
- `uv run mypy --strict tests/__init__.py` 零错误
- `uv run ruff check tests/__init__.py` 零警告

---

## 范围约束

- **仅** 创建 `tests/__init__.py`
- 严禁创建 `test_*.py` 文件（属于后续 TS-x 任务）
- 严禁修改 `pyproject.toml`
