# B-1: 编写 pyproject.toml

## 任务背景

CodeSense_V1 是最小 MCP 服务。技术栈固定为：

- Python 3.14（Windows）
- 官方 `mcp` Python SDK（stdio 传输）
- `jsonschema` 做参数校验
- `uv` 管理依赖
- 测试用 `pytest` + `pytest-asyncio`
- 静态检查 `mypy --strict`、风格 `ruff`

服务命令行入口：`codesense` → `codesense_v1.server:main`。
项目目录约定（来自概要设计 §6）：

```
CodeSense_V1/
├── pyproject.toml
├── src/codesense_v1/   ← 源码包
└── tests/              ← pytest 测试
```

---

## 实现目标

创建 `pyproject.toml`，作为项目唯一依赖与工具配置入口；执行 `uv sync` 能成功安装；执行 `codesense` 命令路由到 `codesense_v1.server:main`（该模块本任务不实现，但入口声明需就绪）。

---

## 需要实现的文件

- `pyproject.toml`（项目根）

不得创建/修改任何其他文件。

---

## 内容契约

必须包含以下小节，字段名严格匹配：

### `[project]`
- `name = "codesense-v1"`
- `version = "0.1.0"`
- `requires-python = ">=3.14"`
- `dependencies = ["mcp", "jsonschema"]`

### `[project.optional-dependencies]` 或 `[dependency-groups]`（uv 推荐 `[dependency-groups]`）
- `dev = ["pytest", "pytest-asyncio", "mypy", "ruff"]`

### `[project.scripts]`
- `codesense = "codesense_v1.server:main"`

### 构建系统
任选其一（与 uv 兼容即可），并配置包发现指向 `src/codesense_v1`：

- `[build-system]` requires = `["hatchling"]`，backend = `"hatchling.build"`
  - `[tool.hatch.build.targets.wheel] packages = ["src/codesense_v1"]`

### `[tool.mypy]`
- `strict = true`
- `python_version = "3.14"`
- `mypy_path = ["src"]`
- `packages = ["codesense_v1"]`

### `[tool.ruff]`
- `line-length = 100`
- `[tool.ruff.lint] select = ["E", "F", "I", "B", "UP"]`

### `[tool.pytest.ini_options]`
- `asyncio_mode = "auto"`
- `testpaths = ["tests"]`
- `pythonpath = ["src"]`

---

## 验收标准

- 在项目根执行 `uv sync` 退出码 0，无解析冲突
- 执行 `uv run python -c "import tomllib; tomllib.load(open('pyproject.toml','rb'))"` 解析无误
- `uv run mypy --version` 与 `uv run ruff --version` 与 `uv run pytest --version` 均可执行
- 文件本身格式合法 TOML

---

## 范围约束

- **仅** 创建 `pyproject.toml`
- 严禁创建 `.python-version`、`uv.lock`（由 `uv sync` 自动生成）、源码文件、测试文件
- 严禁修改任务范围外的任何文件
