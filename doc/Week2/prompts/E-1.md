# E-1: 实现 src/codesense_v1/errors.py

## 任务背景

`errors` 模块为 L4 基础设施层，集中定义工具调用过程中可能抛出的领域异常类。所有异常最终由 `registry.dispatch`（后续任务）统一捕获并转换为 MCP `CallToolResult(isError=true)` 响应。

依赖关系：本模块**零内部依赖**，是依赖图叶子节点。
- `registry` 会 `except ToolError as e: ...`，读取 `e.message`
- `tools/*` 会 `raise InvalidArgumentError(...)`
- 严禁工具内 raise `ValidationError`（专属 registry 校验阶段）

---

## 实现目标

定义 3 个异常类构成的继承体系：

```
Exception
  └── ToolError              基类，含 message 属性
        ├── ValidationError       schema 校验失败（registry 抛）
        └── InvalidArgumentError  业务级非法参数（tools 抛）
```

## 接口契约

```python
class ToolError(Exception):
    """工具领域异常基类。所有可预期的业务/校验错误都应继承自此类。"""

    def __init__(self, message: str) -> None: ...

    @property
    def message(self) -> str: ...


class ValidationError(ToolError):
    """JSON Schema 校验失败时抛出。由 registry 在调用 handler 前抛出。"""


class InvalidArgumentError(ToolError):
    """业务级非法参数（schema 校验通过但语义非法），如 NaN / Infinity / 溢出。
    由工具实现内部抛出。"""
```

### 类型注解要点
- 构造参数 `message: str` 必填
- `message` 属性返回 `str`，等价于 `str(exc)`（实现可在 `__init__` 内 `super().__init__(message)` 并存 `self._message = message`，property 返回之；或直接 `return str(self)`）
- **不**引入额外字段（错误码、详情 dict 等暂不需要）

### 错误码与异常处理规范
- **不**使用数字错误码，仅用异常类型区分
- 异常**不得**携带敏感信息（路径、堆栈、内部对象 repr）
- 文案规范（由调用方传入，本模块仅承载字符串，不强制校验）：
  - `ValidationError`：示例 `"参数错误：缺失必填参数 'b'"` / `"参数错误：'a' 期望 number，收到 str"`
  - `InvalidArgumentError`：示例 `"参数错误：'a' 不能为 NaN"` / `"参数错误：结果溢出"`

## 验收标准

- 3 个类均能 `from codesense_v1.errors import ToolError, ValidationError, InvalidArgumentError` 导入
- `issubclass(ValidationError, ToolError)` 与 `issubclass(InvalidArgumentError, ToolError)` 为 True
- `ToolError("x").message == "x"`、`str(ToolError("x")) == "x"`
- `uv run mypy --strict src/codesense_v1/errors.py` 零错误
- `uv run ruff check src/codesense_v1/errors.py` 零警告
- 文件零内部依赖（不 `import codesense_v1.*`）
- 全部公开符号带完整类型注解
