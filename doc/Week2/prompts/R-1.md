# R-1: 实现 src/codesense_v1/registry.py

## 任务背景

`registry` 模块为 L2 注册/分发层，承担两件事：

1. **注册**：提供 `@tool` 装饰器，把每个工具的 `(name, description, input_schema, handler)` 收集到全局表 `_REGISTRY`
2. **分发**：提供 `list_tools()` 与 `dispatch()` 两个对外函数，由 L1 入口 `server` 直接对接到 mcp SDK 的协议回调

`dispatch` 内部固定流程：**取 spec → jsonschema 校验参数 → 调用 handler → 结果包装为 `CallToolResult` → 异常统一转换**。

返回类型完全贴合官方 `mcp.types`，server 层无需二次转换。

### 已有依赖（可直接 import）

- `codesense_v1.errors`：已实现 `ToolError`、`ValidationError`、`InvalidArgumentError` 三个异常类
  - `ToolError(message: str)`，含 `message: str` 属性
  - `ValidationError`、`InvalidArgumentError` 继承 `ToolError`
- 第三方：`mcp.types`（`Tool`、`TextContent`、`CallToolResult`）、`jsonschema`（`Draft202012Validator`）

### 严禁

- import `codesense_v1.tools`（保持依赖单向；tools 反向依赖 registry）
- import `codesense_v1.schemas`（schema 通过参数传入）

## 接口契约

```python
from __future__ import annotations
from dataclasses import dataclass
from typing import Awaitable, Callable, Final, Union

from mcp.types import Tool, TextContent, CallToolResult

ToolHandler = Callable[..., Union[str, Awaitable[str]]]
"""工具函数签名：接收 schema 中声明的关键字参数，返回字符串（同步或异步均可）。"""


@dataclass(frozen=True)
class ToolSpec:
    name: str
    description: str
    input_schema: dict
    handler: ToolHandler


_REGISTRY: Final[dict[str, ToolSpec]] = {}


def tool(
    name: str,
    description: str,
    input_schema: dict,
) -> Callable[[ToolHandler], ToolHandler]:
    """装饰器：将被装饰函数注册为 MCP 工具。

    重复注册同名工具抛出 RuntimeError（启动期错误，非运行期）。
    被装饰函数原样返回，不做包装，便于单元测试直接调用。
    """


def list_tools() -> list[Tool]:
    """返回所有已注册工具的 mcp.types.Tool 列表（用于响应 tools/list）。"""


async def dispatch(name: str, arguments: dict) -> CallToolResult:
    """调用名为 name 的工具，返回 mcp.types.CallToolResult。
    永不抛异常：所有错误转为 isError=True 的结果。
    """
```

### 类型注解要点

- `ToolHandler` 用 `Callable[..., Union[str, Awaitable[str]]]`，允许同步或 `async def`
- `dispatch` 自身 `async`，便于 server 直接 await，且兼容异步 handler
- 返回类型严格使用 `mcp.types.Tool` / `CallToolResult`
- `ToolSpec` 选择 `frozen=True` 的 dataclass，防止注册后被意外修改

## 行为契约

### 注册期（`@tool` 装饰器）

```text
def tool(name, description, input_schema):
    def deco(fn):
        if name in _REGISTRY:
            raise RuntimeError(f"tool '{name}' already registered")
        _REGISTRY[name] = ToolSpec(name, description, input_schema, fn)
        return fn
    return deco
```

- 同名重复注册 → `raise RuntimeError(f"tool '{name}' already registered")`
- 装饰器**返回原函数**（不包装），便于测试直接调用

### list_tools

- 遍历 `_REGISTRY.values()`，构造 `mcp.types.Tool(name=..., description=..., inputSchema=...)`
- 返回 list，顺序不强求

### dispatch 分发期（永不向外抛异常）

| 触发条件 | 处理 | 响应 |
|----------|------|------|
| `name` 不在 `_REGISTRY` | 不抛，直接构造错误响应 | `isError=True`，文案 `"未知工具：'<name>'"` |
| jsonschema 校验失败 | 内部 `raise ValidationError(...)`，本函数 except 转换 | `isError=True`，文案见下表 |
| handler 内 `raise ToolError`（含子类） | except 捕获 | `isError=True`，文案 = `e.message` |
| handler 内任意其他异常 | 兜底 except | `isError=True`，文案 `"内部错误：<exc_type 名字>"`；**不**泄漏堆栈与 repr |

### jsonschema 错误文案翻译表

将 `jsonschema.ValidationError` 翻译为 `codesense_v1.errors.ValidationError`，按 validator 类型生成文案：

| jsonschema validator | 输出文案模板 |
|----------------------|--------------|
| `required` | `参数错误：缺失必填参数 '<field>'` |
| `type` | `参数错误：'<field>' 期望 <expected>，收到 <actual>` |
| `additionalProperties` | `参数错误：不允许的多余参数 '<field>'` |
| 其他 | `参数错误：<jsonschema 原始 message>` |

提取规则：
- `<field>`：`required` 场景从 `error.message` 解析（jsonschema 报 `'b' is a required property`，取引号内字段名）；`type` 场景用 `error.absolute_path[-1]`；`additionalProperties` 场景从 `error.message` 解析或从实例 keys 与 schema `properties` keys 差集推导
- `<expected>`：`error.validator_value`（如 `"number"`）
- `<actual>`：`type(error.instance).__name__`（注意映射：`int`/`float` → 一般报 `type` 不会触发，因为 number 兼容；但若收到 `bool` 之外的非数值会触发，需返回 Python 类型名）

### dispatch 主流程伪代码

```text
async def dispatch(name, arguments):
    spec = _REGISTRY.get(name)
    if spec is None:
        return _error(f"未知工具：'{name}'")

    try:
        _validate(arguments, spec.input_schema)   # 内部 raise ValidationError
    except ValidationError as e:
        return _error(e.message)

    try:
        result = spec.handler(**arguments)
        if inspect.isawaitable(result):
            result = await result
    except ToolError as e:
        return _error(e.message)
    except Exception as e:
        return _error(f"内部错误：{type(e).__name__}")

    return CallToolResult(
        content=[TextContent(type="text", text=str(result))],
        isError=False,
    )

def _error(msg: str) -> CallToolResult:
    return CallToolResult(
        content=[TextContent(type="text", text=msg)],
        isError=True,
    )
```

### 校验器选择

- 使用 `jsonschema.Draft202012Validator(schema)`
- 取第一个错误即可：`next(validator.iter_errors(instance), None)`

### 不变量

- `dispatch` 任何路径返回的 `CallToolResult` 必满足 `len(result.content) >= 1`
- 进程不因 dispatch 而退出
