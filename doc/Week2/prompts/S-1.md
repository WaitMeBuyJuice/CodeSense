# S-1: 实现 src/codesense_v1/schemas.py

## 任务背景

`schemas` 模块为 L4 基础设施层，集中存放所有工具的 JSON Schema 常量。**纯数据模块**，不执行任何校验逻辑。

为什么单独成模块：
- 与工具实现解耦，未来可被文档生成、客户端示例复用
- 便于集中检查 schema 一致性（如 `additionalProperties: false`）

依赖关系：本模块**零内部依赖**，是依赖图叶子节点。

---

## 实现目标

声明 `ADD_INPUT_SCHEMA` 常量，描述 `add(a, b)` 工具的 `inputSchema`，结构遵循 [JSON Schema Draft 2020-12](https://json-schema.org/draft/2020-12)（与 `jsonschema.Draft202012Validator` 一致）。

## 接口契约

```python
from typing import Final

ADD_INPUT_SCHEMA: Final[dict] = {
    "type": "object",
    "properties": {
        "a": {"type": "number", "description": "加数 a"},
        "b": {"type": "number", "description": "加数 b"},
    },
    "required": ["a", "b"],
    "additionalProperties": False,
}
```

### 字段约定（所有工具 schema 必须遵守）

| 字段 | 必填 | 取值 | 说明 |
|------|------|------|------|
| `type` | 是 | `"object"` | 工具入参必为对象 |
| `properties` | 是 | dict | 每个参数一项，含 `type` 与 `description` |
| `required` | 是 | list[str] | 所有必填参数名 |
| `additionalProperties` | 是 | `False` | 禁止多余参数（对应需求 FR-5） |

### 类型注解要点

- 使用 `Final[dict]` 标注，禁止运行时修改
- 命名约定：`<TOOL_NAME_UPPER>_INPUT_SCHEMA`（本任务仅 `ADD_INPUT_SCHEMA`）

## 验收标准

- `from codesense_v1.schemas import ADD_INPUT_SCHEMA` 成功
- `ADD_INPUT_SCHEMA["type"] == "object"`
- `ADD_INPUT_SCHEMA["required"] == ["a", "b"]`
- `ADD_INPUT_SCHEMA["additionalProperties"] is False`
- `ADD_INPUT_SCHEMA["properties"]["a"]["type"] == "number"`
- `ADD_INPUT_SCHEMA["properties"]["b"]["type"] == "number"`

- `uv run mypy --strict src/codesense_v1/schemas.py` 零错误
